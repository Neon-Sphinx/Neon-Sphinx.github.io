Reposted from skills.sh 排行榜前 10：最受欢迎的 Agent Skills 全解析 by AI一人公司实战

背景：为什么需要 Agent Skills？

这两年 AI 编程工具爆发式增长——Claude Code、Cursor、GitHub Copilot、Windsurf… 每个工具都在快速进化。

但你会发现一个问题：不同的 AI 工具，默认能力差异很大。

比如 Claude Code 擅长系统架构分析，Cursor 更懂上下文补全，Copilot 嵌入 IDE 体验最好。如果你用 Claude Code 开发 React，它可能不会主动遵循 Vercel 的最佳实践；如果你用 Cursor 做 SEO 优化，它可能不知道从何入手。

Agent Skills 就是为了解决这个问题而生的。

它是一个"技能插件系统"——你可以把特定的知识、规则、最佳实践，打包成一个 Skill，让任何 AI 编程工具"加载"这个技能后，就能获得对应的专业能力。

什么是 skills.sh？
#

skills.sh 是一个 Agent Skills 目录，由 Vercel 运营。

你可以把它理解成"Agent 版的 npm"——上面汇集了各种可以增强 AI 编程工具的技能包。

截至 2026 年 1 月，skills.sh 已经收录了 200+ 个 Agent Skills，覆盖前端、后端、设计、营销、安全等各个领域。

Agent Skills 如何工作？
#

传统 AI 编程 vs 使用 Skills
#

传统方式：

你 → AI 工具 → 通用模型 → 输出（可能不够专业）

使用 Skills：

你 → AI 工具 → 加载 Skill → 专业模型 → 输出（符合特定规范）

一个实际例子
#

假设你用 Claude Code 开发一个 Next.js 应用：

不加 Skill 时：

你：帮我写一个 React 组件

Claude：好的，这是组件代码…（按通用方式写）

加上 vercel-react-best-practices 后：

你：帮我写一个 React 组件

Claude：好的，我会遵循 Vercel React 最佳实践：

  • 使用客户端组件 ‘use client’

  • 避免在客户端组件中直接导入服务器组件

  • 使用动态导入优化代码分割

…（输出符合 Vercel 规范的代码）

如何安装和使用 Agent Skills？
#

方式一：命令行安装（推荐）
#

npx skills add vercel-labs/agent-skills --skill "vercel-react-best-practices"

安装后，该技能会自动被你的 AI 工具识别。

方式二：在对话中引用
#

大多数 AI 编程工具支持在对话中直接引用 Skill：

你：请使用 @vercel-react-best-practices 帮我审查这个组件

AI：好的，我会按照 Vercel React 最佳实践来审查…

方式三：项目级配置
#

你可以在项目根目录创建配置文件，指定项目默认使用的 Skills（具体格式因工具而异）。

Claude Code 中的使用技巧
#

Claude Code 对 Skills 的支持非常友好：

  1. 全局安装

    ：使用 npx skills add 安装的 Skill 会被 Claude Code 自动识别

  2. 对话引用

    ：你可以在对话中直接引用技能名称

  3. 查看已安装 Skills

    ：npx skills list

  4. 移除 Skill

    ：npx skills remove <skill-name>

前 10 Skills 排行榜
#

1vercel-react-best-practices（59.3K 安装）

**分类：**前端开发React

**来源：**Vercel 官方

为什么第一？

Vercel 是 Next.js 的开发公司，他们的 React 最佳实践是经过大规模生产验证的。60K+ 安装量说明这个技能几乎是 React/Next.js 开发者的"标配"。

功能：

提供 React 和 Next.js 的最佳实践指南，包含 40+ 条优化规则，涵盖：

  • 性能优化

    ：代码分割、懒加载、图片优化

  • 代码规范

    ：组件设计、状态管理、Hooks 使用

  • 架构模式

    ：服务端组件 vs 客户端组件、数据获取策略

  • 部署优化

    ：Vercel 平台特定的优化建议

实际效果示例：

不加 Skill：Claude 可能写出一个通用的 React 组件

加上 Skill 后：Claude 会主动建议：

  • 这个组件应该用 ‘use client’ 吗？

  • 是否需要动态导入来减少初始包大小？

  • 图片是否应该用 next/image 优化？

适合场景：

  • 用 React/Next.js 开发 Web 应用

  • 希望代码符合 Vercel 推荐的标准

  • 团队需要统一的代码规范

  • 追求极致性能的项目

安装：

npx skills add vercel-labs/agent-skills --skill "vercel-react-best-practices"

2web-design-guidelines（44.9K 安装）

**分类：**UI/UX 设计

**来源：**Vercel 官方

为什么第二？

独立开发者往往没有专业设计师支持，这个技能让 AI 变成你的"虚拟 UI 设计师"，帮你避免常见的设计错误。

功能：

包含 100+ 条网页设计和 UI/UX 审查规则，涵盖：

  • 布局原则

    ：视觉层次、信息架构、响应式布局

  • 色彩系统

    ：对比度、色彩搭配、暗色模式

  • 排版规范

    ：字体选择、行高、字间距

  • 可访问性

    ：A11Y 标准、键盘导航、屏幕阅读器支持

  • 交互设计

    ：微交互、动效、反馈机制

实际效果示例：

不加 Skill：AI 可能只关注功能实现

加上 Skill 后：AI 会提醒：

  • 按钮对比度不够，可能影响可访问性

  • 这个表单缺少错误状态反馈

  • 移动端布局需要优化触摸目标大小

适合场景：

  • 开发需要兼顾设计的项目

  • 想让 AI 帮助审查 UI 设计

  • 缺乏专业设计师但希望界面专业

  • 需要遵循可访问性标准

安装：

npx skills add vercel-labs/agent-skills --skill "web-design-guidelines"

3remotion-best-practices（42.5K 安装）

**分类：**视频开发Remotion 框架

**来源：**Remotion 官方

什么是 Remotion？

Remotion 是一个革命性的框架——让你用 React 代码来写视频。想象一下，你可以像写网页一样写视频，实现程序化视频生成、批量渲染、数据驱动内容等。

功能：

提供使用 Remotion 的最佳实践，包括：

  • 性能优化

    ：视频渲染性能、内存管理

  • 代码结构

    ：组件复用、组合模式

  • 渲染策略

    ：并发渲染、分片渲染

  • 媒体处理

    ：音频、视频、图片的最佳处理方式

实际效果示例：

不加 Skill：AI 可能写出能跑但性能一般的 Remotion 代码

加上 Skill 后：AI 会建议：

  • 这个视频应该用多个序列并行渲染

  • 使用 composition 的静态数据可以优化渲染速度

  • 音频应该预加载避免卡顿

适合场景：

  • 用 Remotion 开发视频内容

  • 程序化生成视频（如数据可视化视频、个性化营销视频）

  • 需要批量生成相似结构的视频

  • 视频自动化生产流程

安装：

npx skills add remotion-dev/skills --skill "remotion-best-practices"

AI 编程工具的短板之一是"不能上网"。这个技能让 AI 获得浏览器能力，可以打开网页、获取信息、执行自动化操作。

功能：

让 Agent 能够操作浏览器：

  • 网页访问

    ：打开 URL、等待加载、处理重定向

  • 信息提取

    ：获取页面内容、截图、提取结构化数据

  • 自动化操作

    ：点击、填表、滚动、等待元素

  • 测试支持

    ：E2E 测试、视觉回归测试

实际使用示例：

你：帮我分析竞品网站的定价页面

AI（使用 agent-browser）：

  1. 打开目标页面

  2. 截图并分析定价结构

  3. 提取价格信息

  4. 总结定价策略

适合场景：

  • 需要抓取网页内容

  • 自动化浏览器操作（测试、爬虫）

  • 让 AI 能够"上网"查信息

  • E2E 测试自动化

安装：

npx skills add vercel-labs/agent-browser --skill "agent-browser"

  1. 先装核心技能

    ：不要一次装太多，先用最核心的

  2. 明确引用

    ：对话时明确指定使用哪个 Skill

  3. 定期更新

    ：Skills 会持续迭代，定期更新获取最新改进

  4. 关注官方

    ：优先使用官方发布的 Skills（质量更可靠）

常见问题
#

Q：Skills 会影响 AI 的通用能力吗？

A：不会。Skills 是"增强"而非"限制"，AI 仍可以使用其通用知识。

Q：可以同时使用多个 Skills 吗？

A：可以。你可以在对话中引用多个 Skills，AI 会综合它们的建议。

Q：Skills 是免费的吗？

A：skills.sh 上的 Skills 都是免费的。有些 Skills 可能依赖第三方服务（如 agent-browser 可能需要 Puppeteer）。

Q：如何创建自己的 Skill？

A：使用 skill-creator Skill，它会指导你完成创建过程。

Claude Code 和 Cursor 的 Skill 互通吗？

A：大部分是互通的，因为它们使用相同的 Skills 生态系统。但具体使用方式可能有差异。
